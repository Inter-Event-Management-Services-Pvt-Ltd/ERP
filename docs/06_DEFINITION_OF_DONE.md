# Definition of Done

A task is not done until:

- implementation matches documented requirements
- authorization rules are enforced server-side
- database constraints remain intact
- relevant tests exist
- unauthorized access tests exist
- lint and type checks pass
- tests pass
- migrations apply locally
- documentation is updated
- checklist is updated
- changelog is updated
- unresolved risks are recorded
- no secrets are committed
- security checkpoint is reviewed

A phase is not done until:

- every required checklist item is complete
- phase acceptance tests pass
- security gate passes
- rollback notes exist
- a human reviews the result
